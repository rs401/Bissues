 19        allows undesired behavior. The issue raised by a user may be a bug in
 20        the code or a user/environment error (requirements not met, dependency

 23    <p>

 25        user, to resolve the issue. A bug tracker identifies a bug, and allows
 26        communication between developers to resolve the bug. A bug/issue tracker
 27        would allow all the developers and users to offer feedback on any
 28        bug/issue.
 29    </p>
 30    <p>

 32        Developers are be able to create new projects and bissues to be tracked.
 33        Users are able to create new bissues for existing projects.

 38        shows all timestamps related to the bissue: date created, date last
